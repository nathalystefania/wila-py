Flujo principal

Configuración

Se guarda en LocalStorage
Nota: Telemetría POST /api/telemetria NO usa JWT; usa X-API-KEY.

1) Crear usuario (Register)

POST /api/auth/register
Si ya existe, la API responde 409 con { "error": "Email ya existe" }.

2) Login

POST /api/auth/login
Al loguear se habilita el resto del flujo y se refresca el panel derecho.

3) Planta

POST/GET/PUT/DELETE /api/plantas
Eliminar planta hace soft-delete y libera asignaciones. No borra telemetría.

4) Motor (dentro de la planta)

POST /api/plantas/{planta_id}/motores
Si te da 409 por duplicado: el codigo ya existe en esa planta.

5) Anillo (dentro del motor)

POST /api/motores/{motor_id}/anillos

6) Carbón (dentro del anillo)

POST /api/anillos/{anillo_id}/carbones

7) Nodo LoRa (registro)

POST/GET/PUT/DELETE /api/nodos
Nota: DELETE falla con 409 si el nodo está ocupado.

8) Asignación nodo ↔ carbón

POST /api/carbones/{carbon_id}/asignar

9) Telemetría

POST /api/telemetria (API KEY) + GET/CSV por carbón (JWT)
Campos del sensor: ax/ay/az + gx/gy/gz + bat.
La consulta cruza historial de asignaciones: el carbón obtiene telemetría del nodo solo dentro de los rangos de asignación.
